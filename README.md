# TeamSpace

## Краткое описание проекта

Проект выполнен в рамках лабораторной работы №3 по курсу «Технологии программирования».  
Разработано одно приложение с общим интерфейсом. Каждая кнопка открывает форму участника с его вариантом задания.

---

## Участники команды и их варианты

| Участник | Вариант | Задание |
|----------|---------|---------|
| **Катя** | 18 | Анализ потребительских расходов (таблица, график, проценты, прогноз) |
| **Лёша** | 17 | Анализ процента детей, рожденных вне брака (таблица, график, проценты, прогноз) |
| **Вика** | 16 | Анализ доли плохих дорог по субъектам РФ (таблица, график, изменения по регионам, прогноз) |

---

## Описание общего интерфейса

Общий интерфейс создан **Катей**.  
Это форма с тремя кнопками, каждая из которых открывает соответствующую форму участника внутри одного приложения.

### Как запустить общий интерфейс

1. Открыть решение `TeamSpace.sln` в Visual Studio  
2. Запустить проект (главную форму)  
3. Нажать на кнопку с именем нужного участника

---

## Описание каждого задания

### Вариант 18 — Потребительские расходы (Катя)

- Загрузка CSV файла с данными о расходах за 15 лет
- Отображение таблицы (год → сумма расходов)
- Построение графика зависимости от года
- Расчёт максимального и минимального процента изменения за год
- Прогноз на N лет методом скользящей средней

### Вариант 17 — Дети, рожденные вне брака (Лёша)

- Загрузка CSV файла с данными о проценте детей, рожденных вне брака
- Отображение таблицы (год → процент)
- Построение графика зависимости от года
- Расчёт максимального и минимального процента изменения за год
- Прогноз на N лет методом скользящей средней

### Вариант 16 — Доля плохих дорог по субъектам РФ (Вика)

- Загрузка CSV файла с данными о доле плохих дорог
- Отображение таблицы (субъект → процент плохих дорог по годам)
- Построение графиков по субъектам
- Определение субъекта с максимальным и минимальным улучшением
- Прогноз на N лет методом скользящей средней

## Инструкция по запуску

### Через Visual Studio

1. Клонировать репозиторий:  
   `git clone https://github.com/KateSkoby/TeamSpace.git`
2. Открыть файл `TeamSpace.sln` в Visual Studio  
3. Собрать решение (`Ctrl + Shift + B`)  
4. Запустить проект `TeamSpace`  
5. Нажать кнопку нужного участника

## Pull Requests

Каждый участник создал Pull Request из своей `feature`-ветки в `dev`.  
После проверки кода коллегами все PR были слиты, затем `dev` слили в `main`.
